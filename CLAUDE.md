## 四、工程规范（每一步都要遵守）

- **版本控制**：全程使用 Git。每完成一个小功能就 commit 一次，提交信息要清晰（说明"做了什么"）。
- **分支**：开发新功能时使用独立分支，完成后再合并到主分支，不要一直在主分支上直接改。
- **项目结构**：HTML / CSS / JS 分文件存放，目录清晰，文件命名统一规范。
- **语义化**：HTML 使用语义化标签（header / nav / main / footer 等），少用无意义的 div。
- **响应式**：使用 media query + flexbox / grid，保证手机和电脑上都正常显示。
- **无障碍（a11y）**：图片写 alt、表单有 label、保证键盘可操作。
- **代码风格**：保持缩进、命名一致；提交前自己先通读一遍。

> 提醒规则：如果我写的代码或提交习惯违反了以上任何一条，请主动指出来。
